1. La vitesse

Le test de vitesse

Images non optimisés.
Images redimensionnés en css (forcées).

Explication:

La note est mauvaise, le site met beaucoup de temps à être téléchargé du serveur (pour l'affichage sur le navigateur).
Il faut en premier temps essayé de réduire le poid des images , les utiliser différement, et réduire le temps de lecture du css...

Travail sur les images

  • Changer le format de certaines images vers des formats plus compressible (.bmp/ .jpg > .jp2)
  • Compresser les images avec un lociciel (jusqu'à 87% pour ce site sur une image)
  • Redimensionner les images à la taille ou elles vont s'affichée (gain de temps pour le navigateur, pas besoin de "réduire l'image une fois téléchargée")

Il faut utiliser le cache du navigateur (non pas pour chargé rapidement la première fois, mais pour mettre dans le cache les parties du site qui ne "changent pas souvant", afin d'accèlérer les prochaines visite.

>> Donc mis en place d'un fichier .htaccess à la racine du site pour dire au serveur qu'il doit mettre dans le cache du navigateur les images.

Compression des pages de rendu

Dans une moindre mesure il faut également reduire le temps de lecture du fichier CSS et JQUERY.
>> Enlever les espaces entre les lignes de code permet de gagner du temps de téléchargement, pour cela on utilise "MINIFY et UNMINIFY"

2. La structure du site/ accessibilité

Le référencements des pages se fait par des robots qui isent le site pour l'indexer dans le serveur de leur "socièté" (GoogleBot pour Google, OrangeBot pour Orange, BinhBot pour Bing) Il lit la/les page(s) pour trouvé les mots clés, la structure du site.

Constats suite à lecture du code html du site

Absence de fichier sitemaps et robots.txt
>> "Robot ne sait pas ce qu'il doit lire", précision sur les pages à indéxer ou pas (ADMIN)
Balises du code html non optimisé (que des div)
>> Il faut une structure, header, section, article, div, footer
Balise des titres, ordre non respecté
>> Il faut une stucture, H1 puis H2, puis H3 (sinon mettre div)

Manque balise label sur la page2.html
>>page2.html devrait s'appeller contact.html

Il vaut mieux mettre du texte qu'une image (analyse par le robot possible)
>> mise en place d'un BLOCKquote à la place d'une image.
Le retour des images:
>> balise ALT et TITLE doivent être renseigné de façon explicite et unique. (ex: 1.jpg >> prise-de-photo.jp2, alt="prise-de-photo" title="shooting-photo")
